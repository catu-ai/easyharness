# Release v0.2.6

## Goal

Prepare the dedicated release PR for `easyharness` version `v0.2.6` by
updating the repository release version from `0.2.5` to `0.2.6` and running the
focused release checks that prove the tracked `VERSION` file maps to the
intended tag.

This is a lightweight plan because the release slice is intentionally limited
to the version bump and release-readiness validation. It does not change core
runtime behavior, harness state semantics, review/archive/evidence behavior,
release automation, or security-sensitive logic.

## Scope

### In Scope

- Update the root `VERSION` file to `0.2.6`.
- Refresh release-facing docs or examples only if they contain a stale current
  version that would confuse the `v0.2.6` release PR.
- Run focused release/version validation before archive.
- Leave a repo-visible release breadcrumb in the PR body before treating the
  lightweight candidate as ready to wait for merge approval.

### Out of Scope

- Changing release automation, CI behavior, packaging scripts, or Homebrew tap
  publishing logic.
- Creating the `v0.2.6` git tag before the release PR merges to `main`.
- Republishing or repairing any prior release.
- Broad documentation rewrites unrelated to the `v0.2.6` release bump.
